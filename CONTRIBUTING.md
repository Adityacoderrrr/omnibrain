# Contributing Guidelines

Thank you for contributing to OmniBrain.

## General Guidelines

- Keep commits small and focused.
- Write clear and meaningful commit messages.
- Update documentation when adding new features or making significant changes.
- Discuss major architectural or design changes with the team before implementation.

## Commit Message Format

Examples:

- docs: update README
- feat: add search agent skeleton
- fix: resolve retrieval bug
- chore: update project structure

## Pull Requests

- Provide a brief description of the changes.
- Keep pull requests focused on a single task.
- Request review from teammates before merging when applicable.

## Code Quality

- Follow consistent coding practices.
- Avoid committing sensitive information such as API keys or credentials.
- Test changes before submission whenever possible.
